Database Management System

DBMS

Introduction

A Database Management System (DBMS) is software that enables users and applications to store, organize, retrieve, and manipulate data efficiently. It acts as a middle layer between the raw data (in the database) and the users/programs that need to access it.

Instead of storing data in random files, a DBMS organizes data in a structured way so that it can be easily queried, updated, and maintained.

Example:

  • Without DBMS → You store contacts in many text files; searching for a phone number takes forever.
  • With DBMS → All contacts are in a database table; you can quickly search by name or filter by city.

2. Functions of a DBMS

A DBMS performs several core tasks:

1. Data Storage Management

Efficiently stores data in structured formats (tables, documents, etc.).

2. Data Retrieval

Uses queries to search and fetch data quickly (e.g., SQL in relational DBMS).

3. Data Manipulation

Allows adding, updating, and deleting records.

4. Data Integrity

Ensures correctness and consistency of data using rules and constraints.

3. Types of DBMS

  1. Relational DBMS (RDBMS)
  2. NoSQL DBMS
  3. Object-Oriented DBMS (OODBMS)
  4. Hierarchical DBMS
  5. Network DBMS

4.Advantages

  • Centralized control of data.
  • Reduced redundancy (no duplicate storage of the same data).
  • Improved security via access control

5.Disadvantages

  • High setup cost (software + hardware).
  • Complexity in design and management.
  • Requires skilled personnel to operate.
